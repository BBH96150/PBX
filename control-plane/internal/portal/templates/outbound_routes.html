What this page does

When someone here dials an outside number, these rules decide which trunk carries the call. The number is matched against each rule's prefix; the most specific enabled rule wins (e.g. +1415 beats +1 beats a blank catch-all). With no rules configured, calls fall back to your default trunk automatically — so you only need rules here if you want to split traffic across trunks or override the caller ID per destination.

# Paging Broadcast — native iOS/Android shells

Thin [Capacitor](https://capacitorjs.com) wrappers that turn the Paging
Broadcast **PWA** into installable native apps for the App Store / Play Store.
They reuse the exact same web app served by the control-plane — there is no
separate UI codebase. The native layer only adds: a home-screen app icon,
native microphone permission prompts, and store distribution.

```
Browser PWA  ─┐
              ├──  one web app  (control-plane: /admin/broadcast)
iOS shell    ─┤
Android shell─┘
```

By default `capacitor.config.json` points `server.url` at the live PWA
(`https://pbx.tendpos.com/admin/broadcast`), so the shells always run the
current web app — ship web changes without re-releasing the apps. Change that
host for your deployment.

## Prerequisites

- Node 18+
- **iOS:** macOS + Xcode 15+, CocoaPods (`sudo gem install cocoapods`), an Apple
  Developer account for signing.
- **Android:** Android Studio (Giraffe+) + JDK 17.

## First build

```bash
cd mobile/paging
npm install
npx cap add ios        # generates ios/ (gitignored)
npx cap add android    # generates android/ (gitignored)
npx cap sync
```

Then apply the **microphone permission** edits below (required — without them
`getUserMedia` silently fails in the WebView), and open the native project:

```bash
npx cap open ios       # → Xcode: select a team, Run on a device
npx cap open android   # → Android Studio: Run
```

> Recording requires HTTPS (the PWA host already is) and a **physical device** —
> simulators usually have no microphone.

## Required native edits

These live in the generated `ios/` and `android/` folders (regenerated by
`cap add`, so re-apply if you delete them — or commit those folders in your
release fork).

### iOS — `ios/App/App/Info.plist`

Add the mic usage string (App Store rejects mic apps without it):

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Paging uses your microphone to record and broadcast pages.</string>
```

WKWebView on iOS 14.3+ supports `getUserMedia` with that key present; no extra
code needed.

### Android — `android/app/src/main/AndroidManifest.xml`

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" />
```

Android's WebView does **not** auto-grant `getUserMedia` even after the runtime
permission is granted — you must handle `onPermissionRequest`. Add this to
`android/app/src/main/java/.../MainActivity.java`:

```java
import android.os.Bundle;
import android.webkit.PermissionRequest;
import android.webkit.WebChromeClient;
import com.getcapacitor.BridgeActivity;

public class MainActivity extends BridgeActivity {
  @Override
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    this.bridge.getWebView().setWebChromeClient(new WebChromeClient() {
      @Override public void onPermissionRequest(final PermissionRequest request) {
        runOnUiThread(() -> request.grant(request.getResources()));
      }
    });
  }
}
```

Also request the runtime `RECORD_AUDIO` permission once at startup (Capacitor's
`@capacitor/microphone` plugin, or a small `ActivityCompat.requestPermissions`
call), or the WebView grant above will be denied by the OS.

## Updating

- **Web/UI change:** nothing to do — the shells load the live PWA.
- **Native config / icon / Capacitor upgrade:** `npx cap sync`, rebuild, re-release.

## Offline / fully-bundled build (optional)

To bundle the app instead of loading the hosted PWA, remove the `server.url`
block from `capacitor.config.json` and copy a built web bundle into `www/`.
`www/index.html` is currently only an unreachable-server fallback.
